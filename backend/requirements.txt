# Speaker diarization. Pulls in torch + torchaudio (~2GB).
# Requires a HuggingFace access token with access to pyannote/speaker-diarization-3.1.
# Version pins (all because pyannote.audio 3.3.2 has not kept up with the
# ecosystem; bumping pyannote.audio to 3.4+ later removes most of these):
#   - torch/torchaudio <2.4: 3.3.2 references torchaudio.AudioMetaData (removed in 2.4).
#   - huggingface_hub <0.24: 3.3.2 still passes use_auth_token= (removed in 0.24).
#   - numpy <2: torch <2.4 was compiled against numpy 1.x; numpy 2.x crashes it.
#   - matplotlib: 3.3.2 imports it transitively from pyannote.audio.tasks but doesn't
#     declare the dep.
numpy<2
# Loose version range — but the actual GPU/CPU build is selected at install
# time by the index URL. Without a +cuXXX local tag, transitive `pip install`
# calls can silently downgrade a CUDA build to the CPU wheel from PyPI.
# To install the GPU build run, in the venv:
#   pip install torch==2.3.1+cu121 torchaudio==2.3.1+cu121 \
#       --index-url https://download.pytorch.org/whl/cu121
torch>=2.2,<2.4
torchaudio>=2.2,<2.4
huggingface_hub>=0.20,<0.24
matplotlib
pyannote.audio==3.3.2
